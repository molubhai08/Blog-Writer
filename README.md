# AI Blog Machine (Aspire IAS)

A production-grade, multi-agent blog generation dashboard built to generate high-quality, SEO-optimized, and "human-like" articles targeting specific audiences (such as Civil Services UPSC aspirants).

---

## 🏗️ Architecture & Agent Workflows

The application coordinates a cooperative team of **6 specialized agents** communicating through state transitions and Server-Sent Events (SSE) to generate the final blog:

```mermaid
graph LR
    Topic["1. Input Topic"] --> Validator["2. Validate & Research"]
    Validator --> Narrative["3. Create Outline"]
    Narrative --> Writer["4. Section Write & Review"]
    Writer --> Output["5. Create SEO & MCQ Quiz"]
```

### The Agent Team
1. **Topic Validator**: Evaluates inputs to reject gibberish, political bias, or unsupported short prompts.
2. **Researcher**: Executes web queries using Tavily to compile top findings and source URLs.
3. **Narrative Agent**: Groups findings into a 4-section outline, maps the topic to General Studies syllabus papers (GS-I to GS-IV) for UPSC mode, and specifies audience-aware writing guidelines.
4. **Section Researcher**: Conducts focused searches on individual section outlines.
5. **Content Writer**: Drafts section contents using strict stylistic rules (sentence variations, simplicity connectors) from `para.md`.
6. **Reviewer (Sapling Detector Loop)**: Automatically audits draft paragraphs for AI markers. If the human-likeness rating drops below 70%, it fires a critique back to the Writer for a rewrite (capped at 2 retries).

---

## ⚡ Key Features

- **💡 Topic Suggester**: Auto-suggests 5 unique, trending, and syllabus-aligned topics based on target audience to spark creativity and save time, completely avoiding duplicate entries from Supabase history.
- **⚡ Concurrent API Load Balancers**: Load balances API calls round-robin across separate pools of **4 Groq API keys** and **2 Cerebras API keys**, fully preventing 429 rate limit exceptions during parallel generation.
- **📝 Custom Markdown Parser**: Safe browser-side renderer that converts markdown subheadings, bold highlights (`**text**`), bullet lists, and ordered lists into premium typography, working alongside the intra-linking resolver.
- **💻 Agent Workflow Terminal**: A scrolling dark-terminal console integrated directly into the dashboard showing real-time timestamps, color-coded agent identities, and detailed backend status logs as the agent team collaborates.
- **⚡ Staggered Concurrency Engine**: Generates all blog sections concurrently with a 4-second offset delay, reducing blog generation time by 300% (from ~60s down to ~25s) while safely avoiding API rate limits.
- **🔗 Hybrid Semantic Intra-linking**: Scans section content for primary keywords, related terms, and tags generated by the LLM, automatically linking terms back to the section that discusses them in greatest detail.
- **📝 UPSC Mains Practice Sandbox**: Includes a dedicated mock exam section with model answer guidelines, keywords, and structural approaches.
- **🗑️ Direct Blog Deletion**: Delete old or duplicate published posts directly from the home Published Library dropdown or directly inside the header of any published post.
- **Section-Level Controls**: Allows editors to manually run the review loop on an individual section card rather than rebuilding the entire blog.
- **Interactive MCQ Quiz**: Automatically builds a custom 5-question conceptual quiz with instant scoring and robust JSON parsing/salvaging recovery.

## ✍️ Content Writer Prompt Design & Rules

The **Content Writer Agent** (`run_writer`) converts research and structures into human-written articles using a structured generation prompt. Below is an overview of how the prompt writes:

### 1. Active Recall / Gamified Headings
For UPSC audiences, dry passive headings (such as *"Challenges"* or *"Solution"*) are converted into situational, role-based questions:
* *Example:* *"If you are a District Magistrate, how would you use the PWDVA 2005 to protect a victim today?"*
* This prompts readers to actively recall information and think critically.

### 2. Inter-disciplinary Linkage
Narratives connect the primary topic to an adjacent General Studies paper syllabus:
* *Example:* Connecting Green Hydrogen (GS-III) to International Relations (GS-II) via *"Green Hydrogen Corridors"* with the EU/Japan.
* *Example:* Connecting Dowry (GS-I) to Ethics (GS-IV) by discussing the *"Crisis of Conscience"* in educated youth.

### 3. Section Role-Playing Context
The agent determines where a section lies in the narrative flow and applies specialized rules:
* **Introduction:** Forced to start with an attention-grabbing hook (e.g. rhetorical question, real-life story, or shocking fact) to hook readers.
* **Body Sections:** Focuses on argument structures, facts, and syllabus connections.
* **Conclusion:** Concludes with a core takeaway summary and a final analytical question.

### 4. Human-like Stylistic Guidelines (`para.md`)
Every section draft is governed by strict stylistic constraints:
* **Paragraph Length:** Short paragraphs (maximum of 3 to 4 sentences).
* **Scannability:** Strategic bolding of core concepts, numbered lists, and bullet points.
* **Tone Mapping:** UPSC/Professional modes use formal, authoritative voices; General mode maps to direct address ("you"), conversational fragments, and variable sentence lengths.

## 🚀 Setup & Local Execution

### Prerequisites
- Python 3.10+
- Node.js 18+

### 1. Backend Setup
1. Navigate to the backend directory:
   ```bash
   cd backend
   ```
2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Create a `.env` file in the root project directory (see **Environment Variables** below).
4. Run the FastAPI development server:
   ```bash
   uvicorn main:app --reload --port 8000
   ```

### 2. Frontend Setup
1. Navigate to the frontend directory:
   ```bash
   cd frontend
   ```
2. Install npm packages:
   ```bash
   npm install
   ```
3. Run the Next.js development server:
   ```bash
   npm run dev
   ```
4. Access the web dashboard at `http://localhost:3000`.
